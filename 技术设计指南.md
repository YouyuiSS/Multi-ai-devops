# 独立产品核心技术设计指南 (基于 Antigravity)

在完成了前序的“产品设计 (PRD & 交互)”和“UI 设计”之后，就来到了整条 AI 智造流水线上**含金量最高、也是直接决定独立产品生死存亡的环节——技术设计 (Technical Design)。**

在这个阶段，由于我们依然处于 **Antigravity (IDE 工作区)** 之中，最大的红利得以显现：**AI 可以“只读”左手的 PRD 文档体系，直接“右手”生成完美贴合需求的技术架构图、数据库模型和 API 设计。**这种文档级的“上下文直通图（Context Passthrough）”消灭了 99% 的传达偏差。

> **💡 核心原则：丰俭由人，按需设计**
> 对于独立产品而言，**除了“技术栈与架构选型”是必选动作外，后续的“领域建模”和“API 设计”其实应该根据产品的复杂度来决定是否需要做、以及做到什么颗粒度。**
> - 如果只是一个极简的单页面网页工具或者纯前端套壳小应用，完全可以跳过数据库设计和 OpenAPI 编写，拔剑直接写代码。
> - 如果涉及到复杂的表单流转、多角色的状态流转或者未来的多端多服务拆分，那么严格走完领域建模和 API 契约设计，能为您节省后期海量的返工时间。

---

## 一、 技术栈与架构选型 (Architecture Choices)
> **目的：** 确定“用什么语言、什么框架、怎么部署”，避免过度设计，坚持“快糙猛”的 MVP 原则。

1. **设定技术栈基调**：
   在 Antigravity 中唤起 AI，给出一个明确的选型原则约束：
   - “作为独立开发者，我需要一个人全栈维护这个项目。请基于 `项目/产品设计/` 目录下的 PRD 文件，帮我推荐最适合的极简技术栈。原则上优先选择单体架构、支持快速构建的框架（如 Next.js 或是 Spring Boot），以及极低运维成本的 BaaS（如 Supabase / Firebase）。”
2. **生成《架构选型说明书》**：
   经过多轮讨论确定后，请 AI 将最终的架构分层（前端、后端、数据库、存储、部署）绘制成 Mermaid 架构图，并写入 `技术设计/整体技术架构.md` 中。

---

## 二、 领域模型与数据库设计 (Domain Modeling & Schema)
> **目的：** 数据库是一切应用的地基。绝不能在一开始把表建死，一定要和大模型充分推演以涵盖所有的边缘业务。

1. **导入业务黑盒，输出领域实体**：
   - 核心 Prompt 技巧：
     > “请精读 `产品设计/初步产品设计_PRD.md` 和 `用户故事.md`。帮我抽象出这套系统里的核心『领域模型实体（Entities）』。不要立刻给 SQL 代码，请先用文字详尽列出：每一个模型代表什么功能，它们之间是一对一、一对多、还是多对多的关系？是否有冗余？”
2. **多轮博弈与审视字段**：
   - **这是极易被忽略的一步。** 大模型第一次写的表结构往往会漏掉审计字段或关键状态位。
   - 追问：“请检查你生成的订单/用户模型，是否考虑了『软删除标志（is_deleted）』？如果用户发生退款链路，状态枚举值是否足够？高并发下的幂等字段是否已添加？”
3. **输出 DDL 终稿**：
   - 待所有字段拉锯敲定后，最后让 AI 直接输出适用于你选定数据库（如 PostgreSQL 或 MySQL）的建表 DDL 语句。
   - 并将所有表结构字典和生成的 Mermaid E-R 图，固化至 `技术设计/数据库设计模型.md` 中。

---

## 三、 API 设计与 OpenAPI 规范 (API Design & Contract)
> **目的：** 形成前后端分离开发的跨语言锲约。这是将来我们让 AI 独立编写前端或后端代码时唯一的准则。

1. **批量生成接口文档**：
   - “根据刚才确定的数据库设计和 UI 设计交互，请列出这版 MVP 需要的所有 RESTful API 接口。涵盖所有的 CRUD 以及复杂的业务鉴权动作。”
2. **规范化输出为 OpenAPI 格式**：
   这是让 AI 的输出具有“工程级”威力的神来之笔。
   - “请将上述确定的所有 API 接口，转化为严格符合 **OpenAPI 3.0 (Swagger) 格式的 yaml 或 JSON** 代码。”
   - 这样做的好处是，这份标准的 OpenAPI 契约可以直接用于后续生成前后端类型代码（Types/Interfaces），甚至可以直接丢进 Postman 中生成测试用例。
3. **文档落地**：
   - 把这些接口定义统统落款至 `技术设计/OpenAPI规范合集.md`。

> _“请读取 `技术设计/数据库模型.md` 和 `OpenAPI规范.md`，帮我用 Spring Boot / Next.js 生成上述所有的 Controller 和 Service 基础代码！”_
