# 独立产品核心开发阶段指南 (基于 Antigravity)

在很多基于 Claude 体系的 AI IDE 中，开发者习惯于使用 `CLAUDE.md` 或者 `.cursorrules` 来作为全局的“系统提示词（System Prompt）”。

如果在 Antigravity 里只寻找这两个文件名，你可能会以为它缺乏规范能力。但实际上，Antigravity **原生内建了功能极其强大的 Rules 机制**（完美降维打击 CLAUDE.md），并在此基础上提供了更高级的**专属技能库 (Skills)** 与 **工作流编排 (Workflows)**。

这套三位一体的机制，让**规范驱动开发（Convention-Driven Development）**在 Antigravity 中变得精准且高度自动化。

---

## 核心机制详解

### 1. 原生的 Rules 约束 (完美替代 CLAUDE.md)
Rules 是纯粹的 Markdown 文件格式（不超过 1.2万个字符），专门用于定制你的技术栈约束和代码风格。你可以随时通过编辑器 Agent 面板右上角的 `...` 菜单中配置 Customizations。

Antigravity 的 Rules 体系远比竞品“一个全家桶文件走天下”的模式更加立体，分类和触发逻辑极为出彩：
*   **全局规则 (Global Rules)**：存放在 `~/.gemini/GEMINI.md`，跨你电脑上的所有项目生效。写在这里的通常是你私人的代码偏好、对 AI 语气的要求等大一统习惯。
*   **工作区规则 (Workspace Rules)**：存放在项目根目录的 `.agent/rules/` 文件夹下。写在这里的是本项目的框架约定和业务规范。

**极其灵活的四种触发策略（Activation）：**
*   **Manual (手动)**：仅在你聊天时用 `@提请该规则名称` 时才生效，免得污染日常对话。
*   **Always On (常驻)**：全局强制生效（也就是 `CLAUDE.md` 扮演的角色）。
*   **Model Decision (按需决策)**：基于你写的一段自然语言描述，大模型自己判断当前对话（写后端接口还是改前端样式）需不需要自动把这条规则加载进脑子里。
*   **Glob (按路径匹配)**：仅在修改特定文件（比如 `src/**/*.ts`）时才自动触发这条规则。这完美解决了“前后端全栈项目里规范互相打架”的问题！

*(高阶玩法：由于 Rules 可以使用 `@文件路径` 直接引用项目里的现成代码作为范例，你可以轻易把你项目里最完美的那份 Controller 文件挂载到“后端规范规则”上！)*

### 2. 专属技能库 (Skills)
*   **存放路径**：`项目根目录/.agents/skills/<技能名称>/SKILL.md`
*   **适用场景**：封装复杂的领域长操作规范，赋予 AI 独立的跨度工作能力。例如建立 `Stitch UI Design` 技能，规定了每次必须读取 PRD -> 分配项目参数 -> 生成代码库这一系列不可侵犯的 SOP 动作。

### 3. 标准工作流 (Workflows)
相比于零散的规则文本，Antigravity 支持更具执行力的**步骤化命令工作流**。
*   **存放路径**：`项目根目录/.agents/workflows/<工作流名称>.md`
*   **杀手级功能 (`// turbo` 自动化)**：在执行步骤（纯跑终端命令环节）上方标注 `// turbo`。AI 遇到该环节会**绕过用户的二次确认弹窗**，全自动为您静默执行任务。这对于一键构建、测试、代码扫描来说堪称神器。

## 如何在 Antigravity 中落地“规范驱动”？

1. **基础盘交给 Rules**：把关于代码规范、库引用的要求分门别类写到 `.agent/rules/` 目录下。针对后端的写一个 `backend-rules.md` (设为 glob 匹配 `*.java`)，前端写一个 `frontend-rules.md` (设为 glob 匹配 `*.tsx`)。让环境隔离自然发生。
2. **重复劳作交给 Workflows**：把提交流程、日常编译排错等固化成一系列 Markdown 设在 `.agents/workflows/` 下，配合 turbo 享受自动跑批。
3. **高难度复杂任务包装为 Skills**：将跨越多个工具调用、信息抽取逻辑的动作打包，作为私人的 Agent 助理技能。

## 推荐的 AI 驱动型项目目录结构 (AI-Native Project Structure)

为了让 Antigravity 能够最精准地读取上下文，同时保持独立开发者全栈项目（通常采用 Monorepo）的整洁，建议采用以下“文档与代码同构”的常用目录规范：

```text
/your-indie-product
├── .agent/                  # 🤖 AI 研发大脑配置区 (规则、工作流)
│   ├── rules/               # 局部代码规范 (如 frontend.md, backend.md)
│   ├── skills/              # AI 专属技能组 (如 code-review.md)
│   └── workflows/           # 自动化工作流 (如 deploy-prod.md)
├── docs/                    # 📄 AI 的上下文之源 (真理唯一)
│   ├── 01_产品设计/          # 存放 PRD、用户故事、功能清单
│   ├── 02_UI设计/            # 存放 Stitch 设计规范/切图参考
│   └── 03_技术设计/          # 存放 架构图、数据库 DDL、OpenAPI.yaml
├── apps/                    # 💻 源代码区 (业务逻辑主程序)
│   ├── web/                 # 前端应用 (例如 Next.js / Vue3)
│   ├── api/                 # 后端服务 (例如 Spring Boot / NestJS / Go)
│   └── mobile/              # 移动端应用 (可选，如 React Native / Flutter)
├── packages/                # 📦 共享物料区 (可选，全栈项目强烈推荐)
│   ├── shared-types/        # 前后端共享的类型定义 (如 OpenAPI 自动生成的 Types)
│   ├── ui-components/       # 基础 UI 组件库 (内部复用)
│   └── core-utils/          # 核心算法与工具函数
├── scripts/                 # 🛠️ 运维与辅助脚本 (数据库迁移、Docker 构建、CI/CD)
├── docker-compose.yml       # 基础设施一键启动 (DB, Redis, MQ 等)
└── README.md                # 项目导航与大盘入口
```

**结构优势解析**：
*   **上下文精准隔离**：当 AI 写前端逻辑时，你可以精准地仅将 `apps/web/` 目录和 `docs/03_技术设计/OpenAPI.yaml` 挂载给它，避免庞杂的后端代码干扰它的注意力，导致“幻觉”。
*   **真理唯一法则**：所有由前序环节（产品策划、技术设计）产出的文档全部集中收敛在 `docs/` 下。这是 AI 编写代码时唯一的“需求与技术法则”，改需求先改文档，彻底杜绝代码和文档脱节。
*   **规范就近约束**：`.agent/` 下的规则库和业务代码放在同一个 Git 仓库。这意味着不论你在哪台电脑上拉取代码，都能获得完全一致的 AI 编码行为约束。

## 结语

在 Antigravity 中，你完全不需要死磕单一且常常因内容过百行而被 AI 忽略的 `CLAUDE.md`。“智能分发上下文”才是这个 IDE 的最优解。

凭借 **智能匹配的 Rules、固化认知的 Skills 以及编排任务的 Workflows** 这套工业级的组合拳，你的独立产品开发将如同拥有一支纪律严明且分工明确的自动化研发团队在为您保驾护航。
